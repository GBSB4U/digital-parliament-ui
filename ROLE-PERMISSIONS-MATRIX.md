# GBSB4U Digital Parliament UI – Role Permissions Matrix (v1.0)

## Purpose
This document defines UI-level permissions for each role.
It ensures visibility without unauthorized power.

## Roles Defined
- Citizen
- Moderator
- Reviewer / Council Member
- Observer

---

## 1. Citizen
### Can View:
- Public debates
- Proposal summaries
- Proposal status
- Decision outcomes
- AI summaries (read-only)

### Can Do:
- Participate in permitted debates
- Submit feedback where allowed

### Cannot:
- Create or approve proposals
- Access reviewer reasoning
- See private deliberations
- Trigger AI analysis directly

---

## 2. Moderator
### Can View:
- All debates
- Moderation logs
- Rule violation flags

### Can Do:
- Structure discussions
- Enforce debate rules
- Flag violations

### Cannot:
- Influence decisions
- Approve or reject proposals
- Modify proposal content

---

## 3. Reviewer / Council Member
### Can View:
- Full proposal text
- Supporting data
- AI analysis outputs
- Debate summaries
- Decision history

### Can Do:
- Record human reasoning
- Request clarifications
- Participate in decision processes

### Cannot:
- Hide decisions
- Override recorded accountability
- Delegate decisions to AI

---

## 4. Observer
### Can View:
- Public dashboards
- Final decisions
- Transparency metrics

### Can Do:
- Nothing interactive

### Cannot:
- Participate in debates
- Submit feedback
- Access internal reasoning

---

## UI Safeguards
- No role has unilateral decision power
- AI outputs are read-only for all roles
- All actions are logged and traceable
- Constitution and Core links visible on every screen
