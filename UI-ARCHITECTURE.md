# GBSB4U Digital Parliament UI – Architecture (v1.0)

## 1. Design Philosophy
The UI is a constitutional interface, not a power center.
It visualizes governance without controlling it.

## 2. Primary User Roles
- Citizen (view, participate in debate where permitted)
- Moderator (structure, enforce rules, no decisions)
- Reviewer / Council Member (review proposals, record reasoning)
- Observer (read-only transparency access)

## 3. Core Screens (v1.0)

### 3.1 Home / Dashboard
- Parliament overview
- Active debates
- Proposal status summary
- Transparency metrics

### 3.2 Debate Room
- Topic-based structured discussions
- Moderation indicators
- AI summaries (read-only)
- Clear rule display

### 3.3 Proposal Viewer
- Proposal text
- Lifecycle stage
- Rationale and supporting data
- Decision history (traceable)

### 3.4 Decision Trace
- Who reviewed
- What reasoning was recorded
- What AI input was considered
- Final outcome

### 3.5 AI Insight Panel
- Summaries
- Scenario analysis
- Risk flags
- Explicit disclaimer: “AI has no authority”

## 4. Governance Safeguards
- No voting buttons in UI
- No AI recommendations labeled as decisions
- Human accountability always visible
- Constitution and Core rules linked on every page

## 5. Future Extensions (Not in v1.0)
- Mobile apps
- Multilingual interface
- Public dashboards
